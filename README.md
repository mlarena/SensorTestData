# Sensor Test Data API

Эмулятор данных дорожных и метеорологических датчиков на базе .NET 10. Предназначен для тестирования интеграций и систем мониторинга без необходимости подключения реального оборудования.

##  Основные возможности
- Генерация реалистичных тестовых данных "на лету".
- Поддержка различных типов датчиков (DSPD, IWS, DOV, DUST, MUEKS).
- Встроенный мониторинг (Prometheus) и документация (Swagger).
- Работа на разных портах для разделения трафика API и метрик.

## Технологический стек
- **Runtime**: .NET 10
- **API**: ASP.NET Core Web API
- **Метрики**: prometheus-net
- **Документация**: Swagger/OpenAPI

## API Endpoints
Все ответы возвращаются в формате JSON с использованием `snake_case`.

- `GET /api/Sensors/DSPD` — Датчик состояния дорожного покрытия.
- `GET /api/Sensors/IWS` —  Датчик комплекный параметров атмосферы.
- `GET /api/Sensors/DOV` — Датчик видимости.
- `GET /api/Sensors/DUST` — Датчик пыли.
- `GET /api/Sensors/MUEKS` — Устройство управления (v1/v2 (	Датчик температеры грунта)).

Swagger UI доступен по адресу: `http://localhost:5055/Swagger` (корень приложения).

## Мониторинг и метрики
Приложение настроено на экспорт метрик в формате Prometheus на отдельном порту.

- **Адрес метрик**: `http://localhost:9102/metrics`

### Собираемые данные:
1. **HTTP метрики**: Количество запросов, длительность обработки, коды ответов (2xx, 4xx, 5xx).
2. **.NET Runtime**: Состояние сборщика мусора (GC), использование памяти, количество потоков.
3. **Системные метрики**: Загрузка CPU, использование диска, сетевая активность хоста.

## Запуск и развертывание

### Локальный запуск
```bash
dotnet run
```
Приложение будет слушать порты `5055` (API) и `9102` (Метрики) согласно `launchSettings.json`.

### Развертывание на Debian (Linux)
Для создания системного сервиса используйте готовый скрипт:
```bash
sudo ./scripts/create-service-sensor-test-data.sh
```
Скрипт настроит `systemd` юнит, который автоматически запускает приложение и пробрасывает необходимые порты.

## Конфигурация
Настройки портов в продакшене можно переопределить через переменную окружения:
`ASPNETCORE_URLS="http://*:5055;http://*:9102"`
